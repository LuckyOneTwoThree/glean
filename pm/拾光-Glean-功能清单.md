# 拾光 / Glean — 功能清单

> AI 驱动的个性化资讯简报应用 | 功能优先级清单

---

## P0 — MVP 核心（9 项）

最小可用版本，验证核心价值：**采集 → AI 分析 → 简报展示 → 导出**。

| # | 功能 | 说明 | 备注 |
|---|------|------|------|
| 1 | 多源资讯采集 | RSS/Atom 订阅 + 第三方新闻 API | 预置主流科技源，支持用户增删 |
| 2 | 资讯入库与去重 | 标题文本相似度去重（Jaccard + 编辑距离），同事件多源合并 | 本地 SQLite 存储；纯规则引擎，不依赖 embedding |
| 3 | AI 质量评分 | 来源可信度 + 信息密度（二维）；支持两种模式：① 本地模式（规则引擎，零成本）② LLM 模式（API 调用，更精准），用户可配置切换 | MVP 先做二维，后续扩展为四维 |
| 4 | 简报配置 | 关注方向、数据源、国内外比例、每日总量 | Onboarding 引导 + 随时可改 |
| 5 | 每日简报 | 研报式结构，附原文链接 + 来源；支持手动生成（首次配置后/未按时生成时）和重新生成；生成过程显示进度状态 | MVP 单档，不做三档；手动生成不发通知 |
| 6 | 资讯列表排序 | 默认按价值分排序 | 也支持按时间排序 |
| 7 | 多粒度摘要 | 一句话摘要 + 三要点展开；支持两种模式：① 本地模式（抽取式，取 meta/首句 + TextRank）② LLM 模式（生成式，质量更高），用户可配置切换 | 两层，不做完整摘要层 |
| 8 | 数据导出 | JSON + Markdown 格式 | 支持单篇导出 + 简报整体导出 + 全量数据打包导出 |
| 9 | LLM 提供商配置 | API Key、模型选择、Base URL；选择省钱模式时可跳过；配置页提供连通性测试 | Onboarding 第⑤步引导配置 |

## P1 — MVP 增强（15 项）

在 MVP 基础上提升体验和可靠性。

| # | 功能 | 说明 | 依赖 |
|---|------|------|------|
| 10 | 文章收藏 | 收藏/取消收藏标记，收藏列表页；支持从简报和资讯列表中收藏 | — |
| 11 | 已读/未读状态 | 文章卡片标记已读状态（进入详情页自动标记），列表支持筛选未读 | — |
| 12 | 评分解释面板 | 点击查看评分依据，文字展示 | 依赖 #3 |
| 13 | 行动建议标签 | 每条洞察附 AI 生成的行动建议 | 依赖 #3 |
| 14 | RSS Feed 输出 | 高质量内容输出为标准 RSS Feed | 依赖 #3 |
| 15 | 数据生命周期管理 | 可配置本地保留天数，超期清理前提醒导出；收藏文章不清理 | 依赖 #2, #8 |
| 16 | AI 运行模式管理 | 省钱模式（本地规则 + 抽取式摘要，零成本）/ 质量模式（LLM 评分 + 生成式摘要，按量计费）/ 混合模式（评分本地 + 摘要 LLM，推荐）；含 API 成本监控与预算上限 | 依赖 #3, #7, #9 |
| 17 | 兴趣画像 | 基于阅读历史自动构建，含信息消费结构可视化 | 需要积累阅读数据 |
| 18 | 用户反馈闭环 | 标记"有用/没用"，反向优化评分模型 | 依赖 #3 |
| 19 | 事件时间线 | 同一事件后续报道自动关联，形成时间线 | 依赖 #2 |
| 20 | 数据源冗余与降级 | 单源中断自动切换备用源，失效提醒 | 依赖 #1 |
| 21 | 采集频率与网络策略 | 可配置采集间隔（1h/2h/4h/仅WiFi/手动），控制电量和流量消耗 | 依赖 #1 |
| 22 | 正文抓取降级 | 有全文 RSS 直接使用；无全文时尝试抓取网页正文（readability 解析）；抓取失败标记"仅摘要"，不影响入库 | 依赖 #1, #2 |
| 23 | 执行记录 | 采集/评分/简报生成的执行日志，设置页可查看最近记录和错误 | — |
| 24 | 本地模型引导下载 | 首次使用语义搜索等需要 embedding 的功能时引导下载 bge-m3，显示进度和存储占用；不阻塞 LLM 模式使用 | 依赖 P3 语义搜索 |

## P2 — V2 差异化（6 项）

扩大使用场景，增加自动化和外部集成。

| # | 功能 | 说明 |
|---|------|------|
| 25 | Webhook 推送 | 新高分文章自动推送到微信/飞书/Telegram |
| 26 | 一键待办同步 | 行动建议→Todoist/滴答清单 |
| 27 | 关注/忽略领域 | 用户显式定义偏好，双因子评分优化 |
| 28 | 探索推荐 + 茧房预警 | 定期推送舒适区外内容，消费过于集中时提醒 |
| 29 | REST API | 暴露核心端点，供外部系统调用 |
| 30 | 自定义导出模板 | 用户自定义导出格式和字段 |

## MVP 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Flutter |
| 状态管理 | Riverpod |
| 本地存储 | SQLite (sqflite) |
| 网络请求 | Dio |
| AI 接口 | mimo API（OpenAI 兼容格式）+ 本地规则引擎 |
| 后台任务 | workmanager (Android) / BGTaskScheduler (iOS) |
| 本地通知 | flutter_local_notifications |
